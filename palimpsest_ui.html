What each module does

  • Scene Analysis -- Builds a color/brightness/texture fingerprint of the environment. Used to find videos filmed in the same room or location.
  • Watermark Detection -- Looks for consistent overlays (channel names, bot stamps) in the corners or edges of video frames. Reveals the redistribution chain.
  • Encoding Chain Analysis -- Checks compression artifacts, bitrate ratios, and block boundaries to estimate whether a file is an original recording or a re-encoded copy, and approximately how many generations of re-encoding it's been through.
  • Screen Recording Detection -- Identifies whether a video was screen-recorded from a phone app (like Telegram) rather than captured by a camera. Checks for status bars, navigation bars, phone screen resolutions, and frame rate patterns.
  • Lighting Analysis -- Analyzes brightness and color temperature over time. Can distinguish natural vs artificial lighting and estimate indoor/outdoor/time-of-day characteristics.
  • Audio Fingerprinting -- Extracts the audio track and creates a spectral fingerprint. Can match videos recorded in the same room by ambient sound. Requires ffmpeg installed.

Scene Matching

Find videos that appear to be filmed in the same location based on color, brightness, and texture analysis. Run forensics on your evidence first.

Audio Matching

Find videos with matching audio environments. Same room, same background noise. Requires ffmpeg for audio extraction.

⚡ ENF Analysis

Electrical Network Frequency -- detect the power grid frequency (50Hz or 60Hz) embedded in audio and video to determine geographic region. Court-accepted forensic technique.

How ENF Works

AC power grids oscillate at either 50Hz (Europe, Asia, Africa, Oceania) or 60Hz (Americas, parts of East Asia). This frequency gets embedded in recordings through electromagnetic interference from power lines and through indoor lighting flicker. ENF analysis extracts this signal to narrow geographic region without any metadata.

Help & Tutorial

Everything you need to know about Palimpsest

What is Palimpsest?

Palimpsest is a metadata forensics toolkit. When someone takes a photo or records a video, the file contains hidden data: what camera was used, when it was taken, GPS coordinates, software used to edit it, and more. This tool extracts, analyzes, and organizes all of that data to support investigations.

The name comes from a palimpsest, a manuscript where the original writing has been scraped off and written over, but traces of the old text remain underneath. Just like metadata: you can try to erase it, but traces often remain.

Built by Ephemera. Source available on GitHub.

Getting Started

  • Step 1: Go to File Analyzer and drop a photo or video. Palimpsest will extract everything hidden inside it.
  • Step 2: Review the results. Simple Mode explains each field in plain English. Advanced Mode shows raw technical data.
  • Step 3: Your file is now in the Evidence Library. You can search, filter, and organize it.
  • Step 4: Create Suspects and link evidence to them to build case files.
  • Step 5: Go to Forensic Suite and run batch forensics. Then check Scene Matching and Audio Matching for connections.
  • Step 6: Use Export & Import to generate reports in any format, or import evidence from a teammate's Palimpsest export.

Features Explained

  • File Analyzer -- Drop any image or video. Palimpsest extracts EXIF data, GPS coordinates, camera info, hashes, and checks for signs of metadata stripping. Videos also get 8 frames automatically extracted for visual review.
  • Simple vs Advanced Mode -- Simple mode shows each metadata field with a plain-English explanation of what it means. Advanced mode shows raw data for technical users.
  • Batch Process -- Select a whole folder of files. Each one gets fully analyzed. Great for processing evidence dumps.
  • Compare -- Pick any two evidence items. Palimpsest checks whether they share the same camera, location, date, codec, resolution, or visual similarity.
  • Duplicate Detection -- Finds exact copies (identical SHA-256 hash) and near-duplicates (visually similar via perceptual hashing). Works across videos and images.
  • Evidence Library -- A searchable table of everything you've imported. Filter by type, GPS, or stripping status. Click any row for full details.
  • Timeline -- All dated evidence sorted chronologically. Useful for establishing event sequences.
  • GPS Map -- Files with embedded GPS coordinates plotted on an interactive map.
  • Suspects -- Create profiles with names, aliases, platform info, threat levels, and typed identifiers (Telegram handles, crypto wallets, emails, etc). Link evidence to suspects to build case files.
  • Hash Lookup -- Paste any MD5, SHA-256, or perceptual hash to find matching evidence.
  • Export & Import -- Five export formats: JSON (machine-readable, importable into other Palimpsest instances), PDF (law enforcement handoff), CSV (spreadsheets, shareable with anyone), HTML (standalone dark-themed report, opens in any browser), and Markdown (paste into Discord/docs). Import lets you load someone else's Palimpsest JSON export into your instance. Duplicates are auto-skipped by SHA-256. All exports filterable by suspect.
  • Forensic Suite -- Deep analysis beyond basic metadata. Scene analysis builds environment fingerprints. Watermark detection finds channel stamps and bot overlays. Encoding chain analysis estimates whether a file is original or a re-encoded copy. Screen recording detection identifies Telegram screen grabs. Lighting analysis characterizes the environment. Audio fingerprinting matches videos by ambient sound.
  • Scene Matching -- After running forensics, compares environment fingerprints across your whole library to find videos filmed in the same room or location.
  • Audio Matching -- Compares audio fingerprints to find videos recorded in the same environment. Same background noise = same room. Requires ffmpeg.
  • ENF Analysis -- Electrical Network Frequency. AC power grids oscillate at 50Hz (Europe/Asia/Africa/Oceania) or 60Hz (Americas/parts of East Asia). This frequency gets embedded in recordings through electromagnetic interference and indoor lighting flicker. ENF analysis extracts this signal to narrow geographic region with zero metadata required. Court-accepted forensic science.
  • Metadata Groups -- Groups all evidence by shared metadata fields (comment, encoder, codec+resolution+fps, file size) to map the distribution pipeline. Same comment = same recording tool. Same encoder = same encoding software. Same chain = same batch processing.
  • AI Vision -- Sends extracted frames to Claude, GPT, or Gemini for forensic analysis. The AI identifies power outlet types (narrows country), visible text and language, room features, lighting fixtures, vegetation, and other geographic clues. Shows cost estimate before running. Supports Anthropic, OpenAI, Google, OpenRouter, and custom endpoints.
  • AI Settings -- Configure your AI provider, API key, model, and custom analysis prompt. Tracks total token usage and cost across all analyses.

What is Metadata Stripping?

When someone wants to hide where a photo or video came from, they can use tools to remove ("strip") the metadata. Palimpsest looks for signs this has happened:

  • No EXIF data at all (most cameras always embed it)
  • Camera info present but dates removed (selective stripping)
  • Editing software detected (Photoshop, GIMP, etc.)
  • Modification date different from capture date
  • Minimal video container data (re-encoded to strip)

A stripping warning doesn't prove anything on its own, but it's a red flag worth noting.

What are Perceptual Hashes?

Regular hashes (MD5, SHA-256) change completely if even one pixel is different. Perceptual hashes (pHash, dHash, etc.) create a "visual fingerprint" of what the image looks like. Two photos of the same scene will have similar perceptual hashes even if one was cropped, compressed, or resized.

This is how Palimpsest finds near-duplicates, and it's why the same re-uploaded video can be tracked across platforms even if it's been re-encoded.

Dependencies

Palimpsest needs these Python packages. All are pip-installable, no external DLLs:

  • Flask -- Web framework (runs the local UI)
  • Pillow -- Image EXIF extraction
  • imagehash -- Perceptual hashing (pHash, dHash, wHash, aHash)
  • opencv-python-headless -- Video analysis, frame extraction, thumbnails, forensic vision
  • hachoir -- Video container metadata (creation dates, codecs, encoder info)
  • numpy -- Numerical processing for forensic analysis
  • scipy -- Audio spectral analysis and signal processing
  • ffmpeg -- Audio extraction from video files (install separately, not a Python package). Download from ffmpeg.org
  • reportlab -- PDF report generation (optional)
